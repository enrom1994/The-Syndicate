# Fix: Create Missing Notifications Table

The "Recent Activity" section is empty or failing because the `notifications` table does not exist in your database.

## Instructions
1. Go to the **Supabase Dashboard**: [SQL Editor](https://supabase.com/dashboard/project/giwolutowfkvkcxlcwus/sql/new)
2. **Copy and Paste** the following SQL code:

```sql
-- Create notifications table
create table if not exists public.notifications (
    id bigint generated by default as identity primary key,
    player_id uuid references public.players(id) on delete cascade not null,
    type text check (type in ('attack', 'income', 'job', 'family', 'system', 'market')) not null,
    title text not null,
    message text not null,
    read boolean default false,
    created_at timestamptz default now()
);

-- Enable RLS
alter table public.notifications enable row level security;

-- Policies
create policy "Users can view their own notifications"
    on public.notifications for select
    using ( auth.uid() = player_id );

create policy "Users can update their own notifications (mark as read)"
    on public.notifications for update
    using ( auth.uid() = player_id );

-- Create indexes
create index if not exists notifications_player_id_idx on public.notifications(player_id);
create index if not exists notifications_created_at_idx on public.notifications(created_at);

-- Insert a welcome notification to test
insert into public.notifications (player_id, type, title, message)
select id, 'system', 'Welcome!', 'Welcome to The Syndicate. This is your activity log.'
from public.players
limit 1;
```

3. Click **Run**.
4. Restart your Telegram Mini App.
